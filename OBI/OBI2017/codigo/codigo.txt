Nome do arquivo: codigo.c, codigo.cpp, codigo.pas, codigo. java, codigo. js, codigo.py2 ou

A professora Maryam está tentando construir um código constituído de uma sequência de N strings
de 10 letras minúsculas, $1, 95, 93,..., Sn. Essas strings da sequência serão, no futuro, concatenadas
de diversas maneiras para formar strings maiores. Mas, para que o código seja válido, a sequência de
strings tem que satifazer uma propriedade bastante específica: nenhuma string da sequência pode
ser substring de uma concatenação de duas strings anteriores na sequência. De forma mais rigorosa,
o código será inválido se existirem três inteiros a,b e k, tais que:

elI<a<k<N,I<b<k<N (apodeser igualab); e
e Sy é substring da concatenação Sa Sh.

Por exemplo, o código S = (aaaaaaabbb, yyuudiwwkl, kkfidaaooa) é válido. Mas se adicionarmos
a string aooaaaaaaa no final da sequência, o código resultante, Sº = (aaaaaaabbb, yyuudiwwkl,
kkfidaaooa, aooaaaaaaa;, será inválido, pois 54 é substring da concatenação S3 51.

Dada a sequência de strings, seu programa deve determinar se o código é válido, ou não.
Entrada

A primeira linha da entrada contém um inteiro N, representando o número de strings na sequência.
As N linhas seguintes contêm, cada uma, uma string de 10 letras minúsculas, definindo a sequência
de strings do código.

Saída

Seu programa deve imprimir uma linha contendo a string “ok” caso o código seja válido, ou contendo
a primeira string na sequência que invalida o código. Quer dizer, contendo Sy onde k é o menor
possível tal que Sy seja substring de uma concatenação de duas strings anteriores na sequência.

Restrições
e 1<N< 10000
Informações sobre a pontuação

e Em um conjunto de casos de teste somando 40 pontos, N < 100
